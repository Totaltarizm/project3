# РГР: Автоматизація міграцій бази даних за допомогою Liquibase

 📌 Опис проєкту
Цей проєкт демонструє впровадження системи керування версіями для бази даних SQLite з використанням інструменту Liquibase. Робота реалізована в ізольованому Docker-середовищі (образ `openjdk:11`), що дозволяє уникнути проблем із залежностями Java та забезпечує стабільність розгортання.

 🛠 Технологічний стек
* СУБД: SQLite (`mylaba.db`)
* Інструмент міграцій: Liquibase 4.1.1
* Драйвер: SQLite JDBC Driver
* Середовище: Docker
* Формат журналу змін: XML

 🚀 Основні етапи роботи

# 1. Налаштування середовища
Проєкт було розгорнуто за допомогою Docker. Основні конфігурації (шлях до бази, драйвер, логін/пароль) винесені у файл `liquibase.properties`.

# 2. Керування станом бази даних
Для роботи з міграціями використовувалися наступні ключові команди:

* Перевірка статусу:
    ```bash
    ./liquibase_bin/liquibase --defaultsFile=./liquibase.properties status
    ```
    *Дозволяє ідентифікувати нові ченджсети до їх застосування.*

* Застосування змін (Update):
    ```bash
    ./liquibase_bin/liquibase --defaultsFile=./liquibase.properties update
    ```
    *Синхронізує XML-опис із фізичною структурою бази даних.*

* Відкат змін (Rollback):
    ```bash
    ./liquibase_bin/liquibase --defaultsFile=./liquibase.properties rollbackCount 1
    ```
    *Демонструє можливість безпечного повернення до попереднього стабільного стану БД.*
